# Cust v0.1 Implementation Notes

Cust v0.1 is intentionally small: it is a safe educational interpreter for a C-like integer subset, written in Rust.

## Architecture

```text
source.c
  -> lexer: characters to tokens
  -> parser: tokens to AST
  -> interpreter: AST execution with integer environment
  -> i64 return value from main()
```

## Supported C subset

- Entry point: `int main() { ... }`
- Additional `int` function definitions: `int name(int param, ...) { ... }`
- Function calls as expressions with integer arguments and per-call local parameter scopes
- Integer and character variables: `int name = expression;`, `char name = expression;`
- One-dimensional arrays: `int xs[3];`, `char text[4];`
- String literals as read-only NUL-terminated byte arrays; they can be indexed and passed to matching array parameters
- Assignment statements and assignment expressions for scalar, array-index, and dereferenced pointer lvalues: `name = expression;`, `array[index] = expression;`, `return x = 1;`, `array[0] = (array[1] = 7);`, and `*p = expression;`. Pointer reassignment remains supported as a statement and in pointer truthiness/equality contexts.
- Array indexing: `array[index]`; pointer indexing for array-backed pointers: `p[index]`.
- Safe first-pass pointers: scalar pointer declarations such as `int *p = &x;`, null as `0`, `*p` reads, `*p = expression;` writes, pointer parameters, array/string decay to pointer arguments, `&array[index]`, `&p[index]`, array-element pointers, pointer truthiness, and pointer equality/inequality for null and supported pointer targets.
- Return: `return expression;`
- Nested block statements with per-block variable scopes and inner shadowing
- Blocks for control flow
- `if (...) { ... } else { ... }`
- `while (...) { ... }`
- `for (...) { ... }`
- `break;` and `continue;` inside loops
- Empty statements (`;`) and expression statements (`expr;`)
- Operators: `+`, `-`, unary `+`, unary `-`, unary address-of `&`, unary dereference `*`, bitwise complement `~`, `*`, `/`, `%`, shifts `<<`/`>>`, comparisons `==`, `!=`, `<`, `<=`, `>`, `>=`, bitwise `&`, `^`, `|`, and logical `&&`, `||`, `!`
- Logical operators use integer truth values (`0` is false, nonzero is true); `&&` and `||` short-circuit and return `0` or `1`.
- Comments: `// line comments` and `/* block comments */`

## Safety boundaries

- Cust interprets source; it does not compile or execute native code.
- Docker runtime runs as a non-root user.
- Compose examples disable networking, drop Linux capabilities, set `no-new-privileges`, and mount examples read-only.
- Interpreter has loop iteration and function call-depth limits to prevent accidental infinite loops and runaway recursion.
- Pointer values are interpreter-owned targets rather than host addresses. Cust reports deterministic diagnostics for null dereference, out-of-scope scalar targets, read-only string storage writes, negative/out-of-bounds array-pointer indexing, unsupported pointer arithmetic, unsupported pointer ordering, and pointer comparisons with nonzero integers.

## Known limitations

Not supported in v0.1:

- Structs, unions, and enums
- Preprocessor directives such as `#include`
- C standard library calls such as `printf`
- Floating-point types
- Richer function compatibility such as declarations/prototypes and `void` functions
- Multiple pointer levels and pointer arithmetic

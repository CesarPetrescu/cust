# Cust v0.1 Implementation Notes

Cust v0.1 is a safe educational interpreter for a growing preprocessor-free C subset, written in Rust. It interprets supported programs directly and returns the integer value from `main()`; native C compilers are used only by tests as external oracles for supported fixtures.

## Architecture

```text
source.c
  -> lexer: characters to located tokens
  -> parser: tokens to AST + type/signature metadata
  -> interpreter: AST execution with scoped interpreter-owned storage
  -> i64 return value from main()
```

## Supported C subset

- Entry points: `int main()` and `int main(void)`.
- Function definitions and prototypes returning `int`, `char`, `void`, one-level pointers, supported structs, or supported unions. Prototypes may use unnamed parameters such as `int add(int, int);` and C-style unsized array spellings such as `void use(int [], struct Point []);`.
- Function calls with scalar, pointer, by-value struct/union, and array-decayed arguments; recursive calls are guarded by a call-depth limit.
- Integer, character, and string literals; strings are read-only NUL-terminated byte arrays.
- Declarations for initialized and default-initialized `int`/`char` scalars, one-dimensional scalar arrays, supported struct/union variables and arrays, enum constants, typedef aliases, one-level pointers, `const` scalar/array/struct bindings, and local persistent `static` storage.
- Top-level `static` on supported globals/functions/prototypes as single-file linkage metadata.
- Typedef aliases for scalar types, named/anonymous structs/unions/enums, block-scoped aliases, pointer aliases, and aggregate typedef definitions with lexical aggregate tag shadowing.
- Supported structs and unions with deterministic interpreter layout: scalar fields, nested aggregate fields, scalar array fields, pointer fields, arrays of supported aggregates, by-value parameters/returns/copy assignment, field lvalues, designated/path/aggregate-array initializers, and one-level aggregate pointers including `->` and aggregate-array pointer indexing/arithmetic.
- Safe pointer model: `int *p = &x;`, `char *p`, `struct Point *p`, `union Number *p`, null as `0`, dereference, reassignment, scalar/array/string/aggregate-array decay, array-element pointers, struct-field pointers, pointer truthiness/equality, bounded array-backed pointer arithmetic/difference, pointer-returning functions, pointee type checks, and const-preserving pointer conversions.
- Assignments and expressions: scalar/array/pointer-index/dereference/field assignment expressions, compound assignments (`+=`, `-=`, `&=`, `|=`, `^=`, `<<=`, `>>=`), prefix/postfix increment/decrement, comma operator, conditional operator, and scalar casts such as `(int)expr`, `(char)expr`, and typedef-spelled scalar casts.
- Control flow: nested blocks with lexical scopes, `if`/`else` including single-statement bodies and dangling-else behavior, `while`, `do ... while`, `for`, `break`, `continue`, and `switch`/`case`/`default` with fallthrough.
- Operators: arithmetic `+ - * / %`, unary `+ - ! ~ & *`, comparisons, logical `&& || !`, bitwise `& ^ |`, shifts `<< >>`, and `sizeof` for supported type and expression operands including const-qualified type contexts and direct aggregate type spellings.
- Comments: `//` line comments and `/* ... */` block comments.

## Diagnostics and tests

- Lexer and parser diagnostics carry 1-based line/column locations; lexer diagnostics for unexpected characters, out-of-range integers, and unterminated block comments include source-line/caret snippets.
- Parser diagnostics include targeted messages for many missing delimiters, separators, assignment operators, names/types, malformed lists, duplicate switch labels, unsupported pointer forms, and other covered malformed programs.
- Test layers include interpreter fixture tests, invalid exact-diagnostic fixtures, deterministic fuzz/safety tests, CLI tests, repository license tests, Docker verification, and C compiler-oracle comparisons for supported fixtures.

## Safety boundaries

- Cust interprets source; it does not compile or execute user programs as native code.
- Docker runtime runs as a non-root user with no network, read-only runtime filesystem, dropped Linux capabilities, and `no-new-privileges`.
- Interpreter loop and call-depth limits guard runaway programs; the CLI also supports `--max-steps N`.
- Pointer values are interpreter-owned targets rather than host addresses. Cust reports deterministic diagnostics for null dereference, out-of-scope targets, read-only/const writes, negative or out-of-bounds indexing, pointer type mismatches, unsupported pointer casts, unsupported pointer ordering, and unsafe const-discard conversions.
- Struct and union sizes/layout are deterministic Cust model decisions, not native ABI/padding promises.

## Known limitations

Cust is still not a full C implementation. Notable unsupported areas include preprocessor directives and includes, the C standard library, floating-point types, multiple pointer levels/pointer-to-pointer storage, native ABI layout compatibility, function pointers, richer casts (including pointer and aggregate casts), declarations in every C grammar position, and many broader C conformance rules outside the documented subset.

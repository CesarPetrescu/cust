# Cust v0.1 Implementation Notes

Cust v0.1 is intentionally small: it is a safe educational interpreter for a C-like integer subset, written in Rust.

## Architecture

```text
source.c
  -> lexer: characters to tokens
  -> parser: tokens to AST
  -> interpreter: AST execution with integer environment
  -> i64 return value from main()
```

## Supported C subset

- Entry point: `int main() { ... }`
- Additional `int` function definitions: `int name(int param, ...) { ... }`
- Function calls as expressions with integer arguments and per-call local parameter scopes
- Integer variables: `int name = expression;`
- Assignment: `name = expression;`
- Return: `return expression;`
- Nested block statements with per-block variable scopes and inner shadowing
- Blocks for control flow
- `if (...) { ... } else { ... }`
- `while (...) { ... }`
- `for (...) { ... }`
- `break;` and `continue;` inside loops
- Empty statements (`;`) and expression statements (`expr;`)
- Operators: `+`, `-`, unary `+`, unary `-`, `!`, `*`, `/`, `%`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `&&`, `||`
- Logical operators use integer truth values (`0` is false, nonzero is true); `&&` and `||` short-circuit and return `0` or `1`.
- Line comments: `// comment`

## Safety boundaries

- Cust interprets source; it does not compile or execute native code.
- Docker runtime runs as a non-root user.
- Compose examples disable networking, drop Linux capabilities, set `no-new-privileges`, and mount examples read-only.
- Interpreter has loop iteration and function call-depth limits to prevent accidental infinite loops and runaway recursion.

## Known limitations

Not supported in v0.1:

- Pointers, arrays, structs
- Preprocessor directives such as `#include`
- C standard library calls such as `printf`
- Floating-point types
- Richer function compatibility such as declarations/prototypes and `void` functions
